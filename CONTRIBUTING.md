# Contributing

## Conventional commits

We enforce Conventional Commits on every commit message.

Format:

```
<type>(optional scope): <description>
```

Examples:

```
feat: add user preferences
fix(auth): handle expired token
chore: update dependencies
```

Allowed types (common):

- feat
- fix
- chore
- docs
- refactor
- test
- perf
- build
- ci
- revert
