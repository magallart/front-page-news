# Front Page News

Front Page News is an Angular + Vercel RSS aggregator focused on editorial home and section views with fast first paint from persisted snapshots and client hydration caches.

## Stack

- Angular 21
- Vercel Functions
- Vercel Blob
- pnpm
- Playwright
- Vitest via Angular test runner

## Requirements

- Node.js 22
- pnpm
- Vercel CLI installed and authenticated for local `vercel dev`

The repository standardizes Node with:

- `.nvmrc` = `22`
- `package.json` engines = `>=22 <23`

## Local Development

## Snapshot and Cache Model

News rendering uses four layers, in order:

1. in-memory query cache
2. persisted IndexedDB snapshot cache
3. persisted Vercel Blob snapshot
4. fresh `/api/news` request

This allows:

- immediate rendering from previously persisted data when available
- background revalidation without clearing the visible UI
- non-blocking refresh indicators and fresh-update banners in hydrated views

## Key Environment Variables

### Public snapshot reads

- `SNAPSHOT_BLOB_BASE_URL`
  - public Blob base URL used to read persisted snapshots

### Snapshot regeneration

- `BLOB_READ_WRITE_TOKEN`
  - Vercel Blob write token for cron regeneration
- `CRON_SECRET`
  - bearer token required by `/api/cron/regenerate-snapshots`

### Optional diagnostics

- `NEWS_PERF_LOGS=1`
  - enables `/api/news` performance/cache logs

## Current Operational Notes

- Base snapshots are regenerated by a protected cron endpoint.
- Snapshot persistence is optimized for Vercel Hobby constraints.
- If Blob or IndexedDB are unavailable, the app should degrade to live API data instead of breaking rendering.
